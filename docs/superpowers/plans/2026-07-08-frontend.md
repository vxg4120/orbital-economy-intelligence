# Orbital Economy Terminal (Frontend) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Read-only FastAPI JSON API + React/Vite/TS SPA ("mission-control terminal") over the
existing identity-graph DB, per `docs/superpowers/specs/2026-07-08-frontend-design.md`.

**Architecture:** `api/` (FastAPI, psycopg via existing `common/db.py`, port 8600) and `web/`
(Vite+React+TS, pnpm, dev proxy `/api`ŌåÆ8600). Tasks F1 (API) and F2 (SPA) are file-disjoint and
run in parallel ŌĆö F2 develops against JSON fixtures matching the spec's DTOs; F3 integrates.

**Tech Stack:** FastAPI, uvicorn, httpx (tests); React 18, react-router-dom, Recharts, strict TS.

## Global Constraints

- The API contract in the spec (`docs/superpowers/specs/2026-07-08-frontend-design.md` ┬¦"API
  contract") is the law for BOTH tasks ŌĆö exact paths, exact field names. Neither task may rename
  a field; contract changes require the controller.
- Read-only: no INSERT/UPDATE/DELETE anywhere in `api/`. Every query parameterized; every list
  endpoint LIMIT-bounded (max 200) with `total` counts.
- BIGINT NORAD ids everywhere; `norad_id` may be NULL (analyst objects) ŌĆö every view must render
  that gracefully (show `ŌĆö`).
- Resolved-status and conflict semantics must match `quality/report.py` (deterministic
  tiebreakers `observed_at DESC, ingest_run_id DESC, source_key`) ŌĆö read it before writing SQL.
- Dark mission-control aesthetic per spec ┬¦"Visual direction". No external network assets (no
  CDN fonts/scripts) ŌĆö system mono stack.
- DB tests: `@pytest.mark.db`, live dev DB (postgresql://oei:oei@localhost:5433/oei), read-only,
  no fixtures inserted.
- Local only: no git remote operations.

### Task F1: FastAPI read-only API (`api/`)

**Files:**
- Modify: `requirements.txt` (add `fastapi`, `uvicorn`), `requirements-dev.txt` (add `httpx`)
- Create: `api/__init__.py`, `api/main.py`, `api/deps.py`, `api/routers/stats.py`,
  `api/routers/satellites.py`, `api/routers/conflicts.py`, `api/routers/operators.py`,
  `api/routers/congestion.py`
- Test: `tests/test_api_stats.py`, `tests/test_api_satellites.py`, `tests/test_api_conflicts.py`,
  `tests/test_api_operators.py`

**Interfaces:**
- Consumes: `common.db.get_conn()`; schema in `db/migrations/*.sql`; conflict/coverage SQL
  patterns in `quality/report.py`; views in `metrics/benchmark_views.sql`.
- Produces: every endpoint of the spec contract, exactly. `api.main:app` importable by uvicorn
  and TestClient. If `web/dist/index.html` exists, mount StaticFiles at `/` (html=True) AFTER
  the api routers so `/api/*` wins.

Steps (per endpoint, TDD): write db-marked TestClient test asserting response shape + a
known-data spot check ŌåÆ run (fail) ŌåÆ implement router SQL ŌåÆ run (pass). Spot checks: NORAD
25544 search returns a row with `cospar_id = '1998-067A'`; `/api/stats` coverage percentages
are floats 0ŌĆō100 and `satellites > 60000`; `/api/conflicts/status` total Ōēź 30; deep object
`/api/satellites/{id}` for the search hit has non-empty `identifiers` and `ownership`.
Implementation notes:
- `api/deps.py`: request-scoped connection dependency yielding `common.db.get_conn()` and
  closing after; rows via `psycopg.rows.dict_row`.
- Search: numeric `q` ŌåÆ `norad_id = %s`; `q` matching `^\d{4}-\d{3}[A-Z]{1,3}$` (case-insensitive)
  ŌåÆ cospar; else `canonical_name ILIKE '%'||q||'%'` ordered by exact-prefix first, LIMIT 20.
- `canonical_status` for search/detail = latest `satellite_status_history` row using the
  deterministic ordering; `operator_name` = current `satellite_operator` (role='owner',
  valid_to IS NULL) joined to `operator`.
- `latest_elements`: latest `gp_elements` row by epoch for the satellite's norad_id (NULL-safe).
- `merge_events`: `merge_log` where surviving_id or merged_id = satellite_id, plus rows whose
  `details` JSONB references it if cheap; LIMIT 50.
- Stale-owners: adapt `quality/report.py`'s section-4 query verbatim semantics.
- Congestion: latest element per norad (`DISTINCT ON (norad_id) ... ORDER BY norad_id, epoch
  DESC`), alt=(apogee_km+perigee_km)/2, LEO only (alt<2000), bins `floor(alt/50)*50` ├Ś
  `floor(inclination/5)*5`, count per bin.
- Final: full `.venv/bin/python -m pytest tests/test_api_*.py -q -W error` green; `ruff` clean.
